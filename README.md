# Automatic Colon Hydrotherapy Management System

A full-stack, HIPAA-aware, offline-capable management system for Colon Hydrotherapy machines.

## Architecture

This system consists of two main components orchestrated by Docker Compose:

1. **Backend / Admin Panel (`/backend`)**
   - Built with React Router v7 (Framework mode).
   - Serves the unified REST API for both the Admin Panel and the Frontend PWA.
   - Connected to MongoDB using Mongoose.
   - Uses server-side rendering for the secure Admin UI.

2. **Frontend PWA (`/frontend`)**
   - Built with Ionic React and Vite.
   - Designed tablet-first (landscape 7"-11" displays).
   - Offline-capable using `vite-plugin-pwa` (Workbox) and `Dexie.js` for IndexedDB storage.
   - Implements an Axios interceptor queue for synchronizing sessions and settings back to the server when connection is restored.

## Key Features

- **Demo Mode Licensing:** New machines start in Demo mode (e.g., 10 sessions). When the limit is reached, the UI locks and prompts the user to contact the supplier.
- **Offline Reliability:** Therapists can conduct sessions, start machines, and log data without an internet connection. Logs are securely buffered locally and synced sequentially upon reconnection.
- **HIPAA Awareness:** Uses JWT HTTP-only cookies and role-based access control to prevent unauthorized exposure of patient or session data.

## Setup Instructions

### Prerequisites
- Node.js 20+
- Docker & Docker Compose

### 1. Environment Configuration

Copy the example environment files:
```bash
cp .env.example .env
```
Edit `.env` to match your local setup.

### 2. Startup Docker Database and Services

To fully leverage the hot-reloading development environment, use Docker Compose Watch:
```bash
docker compose up --watch
```
This will automatically launch the MongoDB database, Backend Node.js process, and Frontend Ionic PWA dev server. It actively traces file changes in your editor and synchronizes them directly into the living containers without restarting everything.

### 3. Accessing the Application

**Frontend PWA**
Simply navigate to: 
`http://localhost:5173`
- Emulate a tablet in landscape orientation in dev tools to experience the true PWA feel.

**Admin Panel**
To view the unified Admin view generated by React Router v7:
`http://localhost:3000/admin`
- **Default Database Seed Login**: 
  - **Email**: `admin@hydrosys.com` 
  - **Password**: `adminpassword123`
- Ensure the database is seeded (`npm run seed` in your backend directory or container) before logging in.

## Offline Sync Architecture

The system achieves robust offline syncing through a dual-store approach:
1. **Zustand State:** Maintains fast, reactive UI states (e.g., timer, machine lock checks).
2. **Dexie DB (`localDB.ts`):** Buffers `sessions` and `settings` mutations locally.
3. **Background Sync:** Workbox queues failed/offline Axios requests and replays them when it detects `navigator.onLine === true`, ensuring data integrity and correct Demo Mode decrementing on the server.
